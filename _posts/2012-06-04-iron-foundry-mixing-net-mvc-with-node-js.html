Generally, the answer would be that you don't have to. However, if you need a light weight, fast, easy to setup web service then a PaaS enabled Node.js Application would be perfect. ASP.NET MVC is great for building out extensive applications, especially if there is complexity and you want to maintain good code integrity through practices like TDD (Test Driven Development) or BDD (Behavior Driven Development). However, sometimes you may want to spin up just a few services, that connect to a Mongo DB, Redis, or some other data source. Node.js is perfect for this for a number of reasons, a few of them include;
  • Node.js is about 1/5th the amount of overhead.
  • Node.js already speaks native JSON, so working with Mongo DB, Couch DB, or Redis is drastically simpler and more direct than using a traditional stack framework that requires extra libraries (like .NET).
  • Node.js is plenty fast. Nuff' said.
